# 功能清单-F04 人物提示词撰写

> 版本：v4.0（默认批量模式：一次处理全部角色，无需逐个调用）
> 状态：**已实现**
> 实现：`src/f04_character_prompt.py`
> 测试：`tests/test_f04.py`
> 依赖：F02（人物元数据提取）、F01（导演视觉基调提取）

---

## 1. 功能说明

基于 **F02 人物元数据（完整输出，含多角色列表）** 和 **F01 导演视觉基调** 的输出，**遍历所有角色**为每个角色生成一条可直接用于 AI 图像生成的最终提示词（`final_prompt`）。

> **调用方式：一次性传入 F02 完整输出 + F01 视觉基调，自动遍历 `characters.list[]` 中全部角色。**

**核心目标**：
- 每个角色输出 **一个场景无关的定妆照提示词**
- 以**中文自然语言填空模板**为统一格式
- 保证同一角色的视觉一致性（通过不可变特征 + 色彩约束）
- **批量处理**：一次调用生成全部角色的提示词，单个失败不影响其他

**设计原则**：
- **一个角色 = 一条最终提示词**，不拆分多版本/多角度/多表情
- **不考虑时间/动作/镜头** —— 这些由下游 F06-B 处理
- **不生成英文版** —— 统一使用中文
- **批量为主、单量为辅**：主入口为批量模式；内部单项函数保留供单独调用

## 2. 输入 JSON

```json
{
  "type": "object",
  "required": ["task_id", "characters", "visual_tone"],
  "properties": {
    "task_id": {
      "type": "string",
      "description": "唯一任务标识符(UUID)"
    },
    "characters": {
      "type": "object",
      "required": ["total", "list"],
      "properties": {
        "total": { "type": "integer", "description": "角色总数" },
        "list": {
          "type": "array",
          "items": { "type": "object", "description": "来自F02的单个角色对象" },
          "description": "角色列表，遍历每个元素生成一条 final_prompt"
        }
      }
    },
    "visual_tone": {
      "type": "object",
      "description": "来自F01的完整视觉基调输出"
    }
  }
}
```

> **数据来源**：`characters` 取自 F02 完整输出的 `characters.list[]`（通常包含多个角色）；`visual_tone` 取自 F01 完整输出。

```json
    }

```

### results[].final_prompt 生成模板

每个 `results[]` 中的 `final_prompt` 由以下内部模板填充而成：

```
请生成一张【name】的形象设计照。整体形象为：【appearance_cn】，穿着【clothing_cn】。
角色的不可变特征包括：【immutable_features（逗号分隔）】。
色彩要求：肤色【skin_tone】，发色【hair_color】，瞳色【eye_color】，主服装颜色【clothing_colors（逗号分隔）】。
风格锚定：【style_anchor】。
质量要求：【quality_tags（逗号分隔）】。
避免出现：【negative_prompt】。
```

## 5. 字段说明

| 字段名 | 含义 | 约束 |
|--------|------|------|
| `task_id` | 批量任务标识符 | 格式 `batch-UUID` |
| `total_characters` | 输入角色总数 | 来自 F02 `characters.list.length` |
| `successful_count` | 成功生成数 | 单个失败不影响其他 |
| `failed_count` | 失败数量 | 失败项的 `results[]` 中含 `error` 字段 |
| `results[]` | 每个角色的生成结果数组 | 每项包含 `character_id` + `final_prompt`（或 `error`） |
| `metadata.generated_at` | 生成时间 | ISO8601 格式 |
| `metadata.source_task_id` | 上游 F02 的 task_id | 用于追溯数据来源 |

## 6. 编写规则

### appearance_cn 撰写规范

**内容顺序**：
1. 种族/地域 → 性别年龄 → 体型身材 → 面部轮廓 → 五官特征 → 发型 → 肤质/特殊标记

**禁止出现**：
- 服装相关词汇
- 姿势/动作描述
- 背景/环境描述
- 质量词（照片级、8K等）
- 光影描述
- 摄影技术参数

### clothing_cn 撰写规范

仅描述服装：款式 + 颜色 + 材质 + 整体风格印象。

### immutable_features 选择标准

选择该角色**视觉识别的核心锚点**，丢失任一都可能导致角色不一致：
- 发型（颜色、长度、基本样式）
- 面部辨识度特征（不是全部面容，而是最有记忆点的）
- 年龄段外观
- 体型轮廓
- 特殊标记（疤痕、胎记等）

### style_anchor 提取规则

从 F01 visual_tone 中提取并转化为中文短语：
- `visual_style.director_style` → 导演名+美学风格
- `color_palette.mood_tone` → 情绪色调
- `atmosphere.overall_mood` → 整体氛围
- `mood_keywords` 中选 2-3 个转为中文

### negative_prompt 必须包含

- 质量类：模糊、低质量、畸变、变形
- 结构类：多余/缺失手指、融合肢体
- 内容类：水印、签名、文字
- 角色专属：防止改变其固有特征（如"改变发色或瞳色"）

## 7. Skill 编写样例

```markdown
# 人物提示词生成（批量）

## 角色
你是一位专业的 AI 绘画提示词工程师，擅长将文字化的人物描写转化为高质量的中文图像生成提示词，确保角色在多次生成中保持视觉一致性。

## 任务
根据提供的角色元数据列表（来自 F02，包含多个角色）和导演视觉基调（来自 F01），**遍历每个角色**，为每个角色生成一条最终的定妆照提示词。

> 注意：你将一次性接收全部角色列表，需逐个为每个角色生成 final_prompt。单个角色的失败不应影响其他角色。

## 输入
1. `characters.list[]`: 角色元数据数组（id, name, gender, age, appearance, clothing, personality, role_type, key_traits）
2. `visual_tone`: 整部作品的视觉基调（F01 完整输出）

## 输出结构
严格输出 JSON：

{
  "task_id": "batch-UUID",
  "total_characters": N,
  "successful_count": M,
  "failed_count": K,
  "results": [
    {
      "task_id": "UUID",
      "character_id": "char_X",
      "final_prompt": "填入模板后的完整中文提示词"
    },
    ...
  ],
  "metadata": { ... }
}

## 每个 results[].final_prompt 模板
请生成一张【name】的形象设计照。整体形象为：【appearance_cn】，穿着【clothing_cn】。
角色的不可变特征包括：【immutable_features逗号分隔】。
色彩要求：肤色【skin_tone】，发色【hair_color】，瞳色【eye_color】，主服装颜色【clothing_colors逗号分隔】。
风格锚定：【style_anchor】。
质量要求：【quality_tags逗号分隔】。
避免出现：【negative_prompt】。

## 关键规则
1. **逐个处理 characters.list[] 中的每个角色**
2. **内部先提取各字段**（name、appearance_cn、clothing_cn、immutable_features、skin_tone、hair_color、eye_color、clothing_colors、style_anchor、quality_tags、negative_prompt），再填入模板
3. appearance_cn 只写外貌（种族→性别年龄→体型→面部→发型→肤质），不写服装/姿势/背景/质量词
4. immutable_features 选3-6条核心识别锚点
5. style_anchor 必须融合至少2个以上F01元素
6. negative_prompt 必须含通用负面词 + 角色专属防变异项
7. final_prompt 是唯一输出，全部使用中文（十六进制色值除外）

## 输出约束
- 只输出合法 JSON 对象，无解释性文字
- 所有字符串不得为空
```

## 8. 与其他功能的关系

```
F02 人物元数据（完整输出，含 characters.list[]）──┐
                                                   ├──→ [F04] → results[].final_prompt
F01 导演视觉基调（visual_tone）────────────────────┘                    │
                                                                      ↓
                                                        下游使用（如人物定妆照生成）

F05 场景提示词 ─────────────────（独立运行，并行）
         │                        ↓
         └──→ 配合 F05 ──→ 人物+场景合成图
```

**调用关系**：
- F04 **一次性接收** F02 完整输出 + F01 视觉基调，内部遍历 `characters.list[]`
- F04 与 F05 **并行独立运行**
- F04 输出与 F05 输出在后端**合成使用**
