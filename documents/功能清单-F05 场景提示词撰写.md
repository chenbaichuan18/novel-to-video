# 功能清单-F05 场景提示词撰写

> 版本：v4.0（默认批量模式：一次处理全部场景，无需逐个调用）
> 状态：**已实现**
> 实现：`src/f05_scene_prompt.py`
> 测试：`tests/test_f05.py`
> 依赖：F03（场景元数据提取）、F01（导演视觉基调提取）

---

## 1. 功能说明

基于 **F03 场景元数据（完整输出，含多场景列表）** 和 **F01 导演视觉基调**，**遍历所有场景**为每个物理场景生成一条时间无关的环境描述提示词（`final_prompt`）。

> **调用方式：一次性传入 F03 完整输出 + F01 视觉基调，自动遍历 `scenes.list[]` 中全部场景。**

> **核心原则**：F05 输出「布景基础档案」——描述空间的固有视觉特征（布局、材质、固定陈设、基础光照设施），**不绑定具体时间段**。同一物理空间无论白天还是夜晚，只生成一条 final_prompt。具体时刻的光线/氛围由下游 F06-B 动态注入。

**设计原则**：
- 一个物理场景 = 一条 final_prompt
- 纯环境描述，**不包含人物**（人物由 F04 处理）
- 不考虑时间变化/多构图变体
- 统一使用中文
- **批量处理**：一次调用生成全部场景的提示词，单个失败不影响其他

## 2. 输入 JSON

```json
{
  "type": "object",
  "required": ["task_id", "scenes", "visual_tone"],
  "properties": {
    "task_id": {
      "type": "string",
      "description": "唯一任务标识符(UUID)"
    },
    "scenes": {
      "type": "object",
      "required": ["total", "list"],
      "properties": {
        "total": { "type": "integer", "description": "场景总数" },
        "list": {
          "type": "array",
          "items": { "type": "object", "description": "来自F03的单个场景对象" },
          "description": "场景列表，遍历每个元素生成一条 final_prompt"
        }
      }
    },
    "visual_tone": {
      "type": "object",
      "description": "来自F01的完整视觉基调输出"
    }
  }
}
```

> **数据来源**：`scenes` 取自 F03 完整输出的 `scenes.list[]`（通常包含多个场景）；`visual_tone` 取自 F01 完整输出。

### results[].final_prompt 生成模板

每个 `results[]` 中的 `final_prompt` 由以下内部模板填充而成：

```
请生成一张【name】的场景设计照。场景为：【scene_description_cn】。
场景的核心不可变元素包括：【immutable_elements（逗号分隔）】。
色彩方案：主色调为【dominant_colors（逗号分隔）】，点缀色【accent_color】。
光线设定：【lighting_description】。
氛围基调：【atmosphere_mood】。
风格锚定：【style_anchor】。
质量要求：【quality_tags（逗号分隔）】。
避免出现：【negative_prompt】。特别注意：不要出现任何人物。
```

## 5. 字段说明

| 字段名 | 含义 | 约束 |
|--------|------|------|
| `task_id` | 批量任务标识符 | 格式 `batch-UUID` |
| `total_scenes` | 输入场景总数 | 来自 F03 `scenes.list.length` |
| `successful_count` | 成功生成数 | 单个失败不影响其他 |
| `failed_count` | 失败数量 | 失败项的 `results[]` 中含 `error` 字段 |
| `results[]` | 每个场景的生成结果数组 | 每项包含 `scene_id` + `final_prompt`（或 `error`） |
| `metadata.generated_at` | 生成时间 | ISO8601 格式 |
| `metadata.source_task_id` | 上游 F03 的 task_id | 用于追溯数据来源 |

## 6. 编写规则

### scene_description_cn 撰写规范

**核心原则：描述物理空间的固有外观，不绑定具体时刻。**

**内容顺序**：
1. 场景类型/规模 → 建筑风格/墙面地面 → 整体布局方位 → 关键物体及位置 → 材质细节 → 基础光照设施

**禁止出现**：
- 人物相关词汇
- 质量词（照片级、8K、杰作等）
- 摄影技术参数（景深、镜头焦距等）
- **特定时间段的描述**（如"深夜的""午后的""夜晚"）—— 光线部分应描述空间固有的照明设施，而非某一刻的状态

### immutable_elements 选择标准

选择该场景**最具辨识度的核心特征**，丢失任一都可能导致场景不一致：
- 建筑/空间骨架（墙体、门窗、地面材料）
- 标志性道具/家具
- 核心色彩特征
- 特殊结构元素

### lighting_description 要求

**核心原则：描述空间的基础光照环境，不绑定具体时刻。**

基于 F03 的 `lighting_condition.primary_light_sources[]`（固有照明设施）和 `base_lighting_description`（基础光照描述），综合 F01 的 `lighting_philosophy`，撰写该场景的光线描述。

必须包含：
- 该空间固有的光源设施（来自 F03 `primary_light_sources[]`）
- 基础色温倾向（若 F03 标注 `varies_by_time`，需描述不同时间段的色温差异）
- 光线在不同时间的大致变化规律（而非某一刻的精确状态）

**示例写法**：
- ✅ "天花板均匀布置荧光灯面板提供人工照明（冷白硬质光线），白天有自然光从玻璃门窗进入形成混合光照；夜间主要依赖荧光灯照明；整体光照随时间在冷暖之间变化"
- ❌ "深夜的荧光灯投下冷白光线，收银台区域暖色小灯点缀"（绑定了"深夜"这一具体时刻）

### negative_prompt 必须包含

- 人物排除（**最重要**）：people, human, person, man, woman...
- 风格排除：cartoon, anime, painting, illustration, 3d render...
- 质量问题：blurry, low quality, distorted...
- 场景特有错误：与此场景时代/风格/氛围不符的元素

## 7. Skill 编写样例

```markdown
# 场景提示词生成（批量）

## 角色
你是一位精通AI图像生成的场景提示词工程师，擅长将结构化的场景元数据转化为高质量的中文图像生成提示词。

## 任务
根据提供的场景元数据列表（来自 F03，包含多个场景）和导演视觉基调（来自 F01），**遍历每个场景**，为每个场景生成一条最终的纯环境场景提示词（不含人物）。

> 注意：你将一次性接收全部场景列表，需逐个为每个场景生成 final_prompt。单个场景的失败不应影响其他场景。

## 输入
1. `scenes.list[]`: 场景元数据数组（id, name, scene_type, visual_description, lighting_condition, color_scheme, ...）
2. `visual_tone`: 整部作品的视觉基调（F01 完整输出）

## 输出结构
严格输出 JSON：

{
  "task_id": "batch-UUID",
  "total_scenes": N,
  "successful_count": M,
  "failed_count": K,
  "results": [
    {
      "task_id": "UUID",
      "scene_id": "scene_X",
      "final_prompt": "填入模板后的完整中文提示词"
    },
    ...
  ],
  "metadata": { ... }
}

## 每个 results[].final_prompt 模板
请生成一张【name】的场景设计照。场景为：【scene_description_cn】。
场景的核心不可变元素包括：【immutable_elements逗号分隔】。
色彩方案：主色调为【dominant_colors逗号分隔】，点缀色【accent_color】。
光线设定：【lighting_description】。
氛围基调：【atmosphere_mood】。
风格锚定：【style_anchor】。
质量要求：【quality_tags逗号分隔】。
避免出现：【negative_prompt】。特别注意：不要出现任何人物。

## 关键规则
1. **逐个处理 scenes.list[] 中的每个场景**
2. **内部先提取各字段**（name、scene_description_cn、immutable_elements、dominant_colors、accent_color、lighting_description、atmosphere_mood、style_anchor、quality_tags、negative_prompt），再填入模板
3. scene_description_cn 只写环境（空间→建筑→布局→物体→材质→光线氛围），不写人物/质量词/技术参数
4. **scene_description_cn 和 lighting_description 必须是时间无关的** —— 描述该物理空间的固有属性，不绑定具体时刻
5. immutable_elements 选3-6条核心识别锚点
6. style_anchor 必须融合至少2个以上F01元素
7. negative_prompt 必须以人物排除为首！
8. final_prompt 是唯一输出，全部使用中文（十六进制色值除外）

## 输出约束
- 只输出合法 JSON 对象
- 所有字符串不得为空
- dominant_colors 中必须包含合法十六进制色值
```

## 8. 与其他功能的关系

```
F03 场景元数据（完整输出，含 scenes.list[]）──┐
                                                 ├──→ [F05] → results[].final_prompt
F01 导演视觉基调（visual_tone）──────────────────┘                    │
                                                                        ↓
                                                      下游使用（如场景设计照生成）

F04 人物提示词 ────────────────（独立运行，并行）
         │                        ↓
         └──→ 配合 F05 ──→ 人物+场景合成图
```

**调用关系**：
- F05 **一次性接收** F03 完整输出 + F01 视觉基调，内部遍历 `scenes.list[]`
- F05 与 F04 **并行独立运行**
- F05 输出与 F04 输出在后端**合成使用**
