## 功能概述

本功能是 **小说转视频流水线的核心汇聚点**，分为两个阶段：

```
阶段A: 文本分段与实体绑定          阶段B: 视频提示词生成
┌──────────────────────┐         ┌──────────────────────┐
│ · 小说文本拆分        │         │ · 基于分段结果        │
│ · 代词消解(他→林默)   │   ──→   │ · 融合F01视觉基调      │
│ · 绑定char_ID+scene_ID│         │ · 内部生成人物/场景描述│
│ · 叙事分析            │         │ → 输出可执行的视频提示词│
└──────────────────────┘         └──────────────────────┘
       输出: segments[]              输出: video_prompts[]
```

**输入来源关系：**

```
小说原始文本 ──────────┐
                       │
:F02 人物元数据 ─────────┤──→ [F06-A] 分段与绑定 ──┐
:F03 场景元数据 ─────────┤                            │
:F01 导演视觉基调 ───────┘                            │
:                                                    ↓
:[F06-A 输出 segments[]] ─────────────────────────────┤
:F01 导演视觉基调 ────────────────────────────────────┼──→ [F06-B] 视频提示词生成
                                                         ↓
                                                   可执行的视频提示词
                                                   (每个segment一组)
```

# 第一部分 (F06-A)：文本分段与实体绑定

---

## 1-A. 功能说明

将**小说全文**按**「场景 × 人物 × 完整动作 × 时长」**四个维度拆分为**连续的视频片段（segment）**，每个片段对应一个可独立生成的视频镜头/场景。同时完成：

1. **代词与别名消解**：将文本中的指代词（"他""她"、称谓"爸爸""妈妈""林先生"等）替换为**规范角色名**
2. **实体ID绑定**：为片段中出现的**人物**绑定 `char_X` ID，为**场景**绑定 `scene_X` ID
3. **时间线标注**：推断并标注每个片段的相对时间位置和预估视频时长

**分段策略（三级优先级，四维约束）**：
- **第一优先级（硬边界）：场景切换** → 地点变化必须拆分 → 保证每个 segment 只有 **1个场景**
- **第二优先级（软边界）：人物动作单元** → **同一场景内**，按「谁+做了什么完整的一件事」拆分 → 保证每个 segment 聚焦 **一组人物的一个完整动作**
- **第三优先级（保护性拆分）：时长控制** → 超过80字或预估>15秒时按子动作切分 → 保证每段视频在 **5-15秒** 可生成范围内

**核心目标**：
- 产出结构化的「拍摄脚本」——每段可直接转化为视频生成提示词
- 解决代词歧义问题，确保后续生成时能准确关联到正确的人物和场景
- 保持原文叙事顺序和完整性

## 5-A. 核心规则详解

### 5-A.1 代词消解规则（最关键）

#### 需要处理的代词/别名类型

| 类别 | 示例 | 处理方式 |
|------|------|----------|
| **第三人称代词** | 他、她、它 | 根据上下文确定指向的角色 |
| **第二人称代词** | 你、您 | 根据「谁在说话→在对谁说话」确定指向 |
| **指示代词作主语** | 这个人、那个女人 | 根据就近先行词确定 |
| **家庭称谓（第三人称）** | 爸、妈、哥哥、姐姐 | 根据说话者和语境确定具体指向 |
| **家庭称谓（提及他人亲属）** | 你爸、他妈、你哥 | 结合说话者和听话者关系链推导 |
| **职业/身份称呼** | 店员、老板、医生、老师 | 绑定到对应的 char_ID |
| **昵称/爱称** | 默儿、小晚、阿林 | 作为别名处理 |
| **所有格代词** | 他的手、她的眼睛、你的声音 | 替换为「角色名的+名词」（如"林默的手"） |

#### 代词消解优先级（从高到低）

```
1. 直接点名 → 无需消解（置信度 1.0）
   例：「林默抬起头」→ 不变
   
2. 同句内有明确先行词 → 高置信度（0.90-0.99）
   例：「苏晚推门进来，她走到货架前」→ 「她」= 苏晚
   
3. 前一句有唯一活跃角色 → 中高置信度（0.80-0.95）
   
4. 对话中的「你」→ 根据说话者确定听话者（0.85-0.98）
   
5. 家庭称谓 → 根据说话者+家族关系图推导（0.85-0.95）
   例：苏晚说「你爸」→ 对象是林默 → 林默的爸 = 林志国
   
6. 多角色同场景时的代词 → 需结合动词/动作语义判断（0.70-0.90）
```

#### 家庭称谓双层推理（最重要！）

遇到「X的亲属称谓」格式时：
1. 先确定 X 是谁（通过前面的消解）
2. 再确定 X 的这位亲属在 characters 中是谁
3. 特别注意「你爸/你妈」这种格式——「你」先消解为某人，再找某人的父母

```
文本: 「苏晚说：『你爸昨天打给我了。』」
消解过程:
1. 识别说话者: 苏晚 (char_2)
2. 识别「你」: 听话者 = 林默 (char_1)
3. 识别「你爸」: 林默的爸爸
4. 查 relationship_context: char_2.refers_to_others_as.char_3 包含「你爸」
5. 确认: char_3 = 林志国
6. 替换: 「你爸」→ 「林志国」
```

#### 特殊情况处理

- **无法确定的歧义代词**：保留原文不替换，记录到 unresolved_pronouns，confidence < 0.50
- **泛指代词**：「一个人」「大家」「有人」等不定向代词 → 不替换
- **间接引语嵌套**：「他说他明天会来」——两个「他」可能指向不同角色

### 5-A.2 分段规则（核心：场景 → 人物+动作 → 时长）

#### 分段策略总览（四维约束）

每个 segment 必须同时满足以下四个维度：

| 维度 | 约束 | 对应优先级 |
|------|------|-----------|
| **场景唯一** | 每个 segment 只绑定 1 个 scene_id | 第1优先级（硬边界） |
| **人物聚焦** | 聚焦同一组人物的行为 | 第2优先级（软边界，隐含在动作单元中） |
| **动作完整** | 每个segment包含「谁+做了什么完整的一件事」 | 第2优先级（软边界） |
| **时长可控** | 字数≤80、时长5-15秒 | 第3优先级（保护性拆分） |

分段按以下**三级优先级**依次判断：

```
第一优先级：场景切换        → 必须拆分（硬边界）     → 保证 1个场景/segment
第二优先级：人物完整动作单元  → 自然断点（软边界）    → 保证 1组人×1个完整动作/segment
第三优先级：单片段时长控制    → 超长时强制切分        → 保证 5-15秒可生成
```

---

#### 第一优先级：场景切换（硬边界，必须拆分）

当以下情况发生时，**必须**创建新 segment：

| 触发条件 | 判定方式 |
|----------|---------|
| **地理位置变化** | 文本中出现新的地点关键词，且与当前 scene_id 不匹配 |
| **空间转移描述** | 「走出便利店」「来到咖啡馆」「回到房间」等明确的空间移动动词 |
| **场景别名匹配** | 文本中的地点词匹配到另一个 `scene_X` 的 aliases |

**示例**：
```
[seg_N]   ...林默站在收银台后...                    ← scene_1 便利店内部
           （文本：「他推开门走了出去」）             ← 场景切换！必须拆分 ↓
[seg_N+1] ...深夜的街道空无一人...                   ← scene_5 便利店门口/街道
```

---

#### 第二优先级：人物完整动作单元（软边界，自然断点）

在**同一场景**内，按**「哪(几)个人 + 做了什么完整的一件事」**来划分。

> **关键约束**：一个 segment 内的场景不变（由第1优先级保证），聚焦的人物组不变，动作是完整的。

##### 动作单元的定义

一个**完整的动作单元**包含（在同一场景内）：

| 组成部分 | 说明 | 示例 |
|---------|------|------|
| **场景** | 已由第1优先级确定，本 segment 内不变 | 便利店·内部（scene_1） |
| **人物组** | 本 segment 聚焦的角色（1人或多人，但组不变） | 林默 / 林默+苏晚 |
| **起因/触发** | 动作开始的原因或前置事件 | 听到风铃声 / 苏晚说话 |
| **动作本身** | 核心行为（可以是复合动作） | 抬头→看→低头→手停住 |
| **结果/状态变化** | 动作结束后的新状态 | 表情凝滞 / 站起身离开 |

##### 动作单元的边界信号

| 边界类型 | 文本特征 | 示例 |
|---------|---------|------|
| **动作完成+状态稳定** | 动作结束后进入静止/等待态 | 「...手顿住了。」→ 后面接的是沉默/环境描写 |
| **新动作发起** | 新的行为开始，与前序动作无直接因果链 | 前一段是「数钱」，下一段是「拿起电话」 |
| **对话轮次切换** | 说话人变更，且话题有实质转换 | A说完→B回应完→A开启新话题 |
| **焦点角色/人物组切换** | 叙事关注点从一个(组)角色的行为转移到另一(组)角色 | 从「林默数钱」→切换到「苏晚独自看货架」 |

##### 不应在动作中途拆分的情况：

- ❌ 一个连贯的物理动作进行中（「他慢慢抬起头，看了她一眼」——这是**一个**动作单元）
- ❌ 一句对话未说完
- ❌ 动作与直接反应之间（提问→回答/动作→即时反应）
- ❌ 情绪递进链中（惊讶→困惑→愤怒的连续升级）

---

#### 第三优先级：单片段时长控制（保护性拆分）

当一个动作单元**过长**时，在保持动作连贯性的前提下进行拆分。

##### 时长标准

| 指标 | 推荐范围 | 强制拆分阈值 |
|------|---------|------------|
| 中文字数 | **15-60字** | > 80 字必须拆 |
| 动作复杂度 | 1-2 个子动作 | > 3 个子动作应考虑拆 |
| 预估视频时长 | **5-15 秒** | > 15 秒必须拆 |

##### 保护性拆分方法

超长动作单元按**子动作节点**拆分，而非按字数机械切割：

```
原始长动作单元（~100字）：
  林默慢慢放下手中的硬币，抬头看了看苏晚，
  又低下头继续数钱。「嗯。」苏晚说：「你爸昨天打电话到我店里来了。」
  他的手顿住了，眼睛微微眯起，然后深吸一口气，
  转过身去整理身后的货架，背对着苏晚站了很久，
  最后低声说了一句「知道了」，然后走向后门的储藏室。

拆分为：
┌─ seg_A ─────────────────────────────────────┐
│ 放下硬币→抬头看→低头继续数钱→「嗯」            │  (~35字, ~10秒)
│ （动作：被动接收信息前的日常行为）              │
└─────────────────────────────────────────────┘
┌─ seg_B ─────────────────────────────────────┐
│ 苏晚说「你爸打电话来了」→ 手顿住→ 眼睛眯起     │  (~25字, ~8秒)
│ （动作：接收消息的情绪反应瞬间）               │
└─────────────────────────────────────────────┘
┌─ seg_C ─────────────────────────────────────┐
│ 深吸气→转身整理货架→背对站立                 │  (~20字, ~12秒)
│ （动作：消化情绪后的回避行为）                 │
└─────────────────────────────────────────────┘
┌─ seg_D ─────────────────────────────────────┐
│ 「知道了」→走向后门储藏室                    │  (~12字, ~8秒)
│ （动作：离开的动作）                          │
└─────────────────────────────────────────────┘
```

**拆分原则（四维约束不变量）**：
1. 在**子动作的自然间隙**处切分（动作转换点）
2. 每个 sub-segment 都要有**明确的视觉内容**可生成画面
3. 拆分后的相邻 sub-segment 通过 `sequence_order` 连续编号关联
4. **四维不变量**：拆分后每个子 segment 仍满足——场景不变、人物组不变、子动作完整、时长≤15秒

### 5-A.3 场景绑定规则

```
对于每个 segment:
1. 扫描文本中的地点关键词
2. 与 scenes[].aliases 进行模糊匹配
3. 检查 time_setting 是否一致
4. 选择 confidence 最高的 scene
5. 若无匹配场景（confidence < 0.5），scene_id 设为 null 并标注原因
```

### 5-A.4 relationship_context 的关键作用

正确的 relationship_context 决定了家庭称谓能否准确消解：

```json
{
  "char_1 (林默)": {
    "referred_by_whom": { "char_2 (苏晚)": ["你"], "char_3 (林志国)": ["儿子"] },
    "refers_to_others_as": { "char_2 (苏晚)": ["苏晚","她"], "char_3 (林志国)": ["爸","老爸","父亲"] }
  },
  "char_2 (苏晚)": {
    "referred_by_whom": { "char_1 (林默)": ["苏晚","她"] },
    "refers_to_others_as": { "char_1 (林默)": ["你"], "char_3 (林志国)": ["你爸"] }
  }
}
```

## 6-A. 质量要求（分段部分）

- **直接点名**的消解置信度必须为 **1.0**
- **高置信度消解**（0.90+）应占总消解数的 **85% 以上**
- **禁止猜测性消解**——宁可标记为 unresolved 也不能错误绑定
- `sequence_order` 必须 1, 2, 3... 连续无跳号
- **每个 segment 必须对应一个可独立生成的视频画面**——不能有无视觉内容的空片段
- **四维约束校验（每个 segment 必须满足）**：
  - ✅ 场景唯一：只有 1 个 scene_binding
  - ✅ 人物聚焦：聚焦同一组人物（characters_present 在 segment 内不变）
  - ✅ 动作完整：包含「谁+做了什么完整的一件事」
  - ✅ 时长可控：15-60字 / 5-15秒；超过80字或>15秒必须保护性拆分
- **场景切换必须正确识别**——这是第一优先级硬边界，漏检会导致后续提示词场景错乱
- **动作单元边界要合理**——不在连贯动作中途拆分，在动作转换点拆分
- **`split_reason` 必须准确标注**——记录该segment是因什么原因被切分出来的（scene_switch / action_boundary / length_split / paragraph_end）
- **保护性拆分的连续性**——被拆分出的子segments之间通过连续的 `sequence_order` 关联
- 所有原文内容必须分配到某个 segment，**不得遗漏**
- text_original 与 text_resolved 的**唯一差异**应是代词/别名的替换

# 第二部分 (06-B)：视频提示词生成

## 1-B. 功能说明

基于 **F06-A 的分段结果**（含角色/场景绑定、别名消解、时间信息）+ **F01 视觉基调**，为每个 **segment** 生成**可直接用于 AI 视频生成工具**的结构化视频提示词。

**核心目标**：
- 将分段文本升级为**动态视频提示词**（增加运动、镜头、时序维度）
- 精确描述每个视频片段中的**人物动作**、**镜头运动**、**场景氛围**和**情绪表达**

## 6-B. 编写规则

### 6-B.1 final_video_prompt 输出格式（v3.0 语义标签格式）

**final_video_prompt 必须使用以下语义标签格式输出**，每段以 `**标签名**：` 或 `【标签名】` 开头：

```

**场景空间**：【场景描述（不含人物）】。
**时代地域**：【时代+地域+阶层氛围】（从 era_setting / world_setting 提炼）。
**人物与动作**：【角色外貌 + 动作链用→串联】。
**镜头语言**：【类型/角度/运动】。
**光线氛围**：【光线描述】。
**情绪弧线**：【起始→过渡→结束】。
**风格锚定**：
- 媒介质感：【medium_style】
- 导演美学：【director_anchor】
- 摄影特点：【cinematography_anchor】
- 参考作品：【reference_anchor】
- 色彩基调：【dominant_colors + accent_color + mood_tone】
- 整体节奏：【pacing_rhythm 体现在镜头/动作密度中】
- 情绪关键词：【mood_keywords 选3-5个融入】
- 光线哲学：【lighting_philosophy 与光线氛围一致】
- 角色设计约束：【character_design_rules 如有则写入】
- 场景设计约束：【scene_design_rules 如有则写入】
**质量要求**：【quality_tags】。
**避免项**：【negative_prompt】。
```

> **格式硬性规则**：每个段落必须以 `**标签名**：` 或 `【标签名】` 格式开头，禁止输出纯连续无分段的长文本。

### 6-B.2 scene_description 编写规则

- 根据场景绑定（scene_binding.scene_name）+ 时间信息（temporal_info）自行生成
- 只写空间→布局→物体→光线氛围（结合具体时刻注入时间相关的光照描述）
- 不写人物（由 characters_in_frame 单独处理）
- 不写质量词（由 quality_tags 处理）
- 控制在 50-100 字

### 6-B.3 时代地域编写规则（新增）

从 `era_setting` 和 `world_setting` 中综合提炼，融入场景或人物描写中：

| 来源字段 | 提炼内容 | 示例 |
|---------|---------|------|
| `era_setting.era` | 时代背景 | "现代""近代""古代""未来" |
| `world_setting.geographic_context` | 地域特征 | "中国南方城市""西藏高原" |
| `world_setting.social_class` | 阶层气质 | "普通市民阶层""知识分子家庭" |
| `world_setting.primary_locations` | 场景类型 | "殡仪馆""老式单元楼" |

输出示例：「现代2020年代中国南方城市普通市民阶层的老旧居住空间」

### 6-B.4 characters_in_frame[].action 编写规则

- 基于 `resolved_text`（代词消解后文本）
- 描述本片段中该角色的**完整动作链**
- 用「→」连接多个子动作，体现先后顺序
- 包含情绪暗示（如"缓慢抬头""骤然停住"）
- 控制在 20-60 字/人

### 6-B.5 camera_shot 选择指南

| 叙事需求 | 推荐 shot_type |
|---------|---------------|
| 环境建立/空间交代 | wide_shot / full_shot |
| 人物互动/对话 | medium_close_up / two_shot / over_the_shoulder |
| 微表情/情绪特写 | close_up / extreme_close_up |
| 动作跟随 | medium_shot + tracking movement |

### 6-B.6 lighting 编写要点

- 光源位置和类型
- 色温（冷/暖/中性）
- 强度感知（明亮/适中/昏暗）
- 特殊效果（阴影、反光等）
- 必须与 F01 的 `lighting_philosophy` 保持一致

### 6-B.7 style_anchor 逐项融合规则（重写——共 13 项硬性要求）

LLM 必须从 visual_tone 中**逐项提取并写入 final_video_prompt**，不允许跳过任何一项：

| # | 来源字段 | 输出到 | 要求 |
|---|---------|--------|------|
| 1 | `visual_style.medium` | 媒介质感 | 如「真人电影质感」「动画风格」，必须在质量要求中体现 |
| 2 | `visual_style.director_style` | 导演美学 | 原样或改写后融入 |
| 3 | `visual_style.cinematography` | 摄影特点 | 指导镜头运动和构图描述 |
| 4 | `visual_style.reference_works` | 参考作品 | 至少提1部，格式如「融合《XXX》的XXX美学」 |
| 5 | `color_palette.dominant_colors` | 色彩基调 | 在场景描述或色彩基调中体现（如「#xxx色系」） |
| 6 | `color_palette.accent_color` | 色彩基调 | 提及出现的位置和方式 |
| 7 | `color_palette.mood_tone` | 色彩基调 | 明确写出整体色调感受 |
| 8 | `atmosphere.pacing_rhythm` | 整体节奏 | 通过镜头运动速度、动作密度来体现 |
| 9 | `atmosphere.overall_mood` | 情绪弧线/风格锚定 | 关键词融入情绪弧线描述 |
| 10 | `mood_keywords` | 情绪关键词 | **从中选取 3-5 个**自然融入各处描述 |
| 11 | `lighting_philosophy` | 光线氛围 | 光线描述必须与其一致 |
| 12 | `visual_constraints.character_design_rules` | 角色设计约束 | 如有相关角色出现，将约束转化为人物外貌/服装/行为描述 |
| 13 | `visual_constraints.scene_design_rules` | 场景设计约束 | 将约束转化为场景的空间/光影/材质描述 |

### 6-B.8 negative_prompt 必须包含

- 质量类：模糊、低质量、畸变、变形
- 风格类：卡通、动漫、绘画、水印、签名
- 视频特有：动作不连贯、面部不一致、过度曝光、帧间抖动
- 人物专属：防止改变角色固有特征（发色/瞳色/服装颜色等）
- 一致性类：跨片段外观不一致、场景元素位置变化

## 7-B. 质量要求（视频提示词部分）

### 7-B.1 final_video_prompt 质量

- **一条完整的中文自然语言提示词**，使用**语义标签格式**输出，可直接复制到视频模型使用
- 长度：**300-600字**（因包含11个语义标签段落，比旧版更长）
- 必须包含（按顺序）：**【DURATION】+ 场景空间 + 时代地域 + 人物与动作(外貌→) + 镜头语言 + 光线氛围 + 情绪弧线 + 风格锚定(13项子标签) + 质量要求 + 避免项**
- 避免**相互矛盾的指令**
- 所有人物名称必须来自消解后的 resolved_text（不能用"他""她"）
- **角色名称格式（硬性规则）**：final_video_prompt 正文中**每次**提到人物时，必须使用「角色名(char_ID)」格式，如 `陈治和(char_2)`、`苏晚(char_1)`。即使多次出现、在动作描述或括号中，也必须始终带 `(char_ID)` 后缀。禁止只写纯名字
- **格式硬性规则**：每个段落必须以 `**标签名**：` 或 `【标签名】` 格式开头，禁止纯连续长文本

### 7-B.2 内部字段质量（用于填充 final_video_prompt）

- 时长必须在 **5-15秒** 范围内
- 出场人物必须**覆盖所有角色**，且 character_id 与 entity_bindings 一致
- 动作描述必须与 resolved_text 对应，不可凭空添加不存在的行为
- style_anchor 必须逐项融合上述 **13 项 F01 元素**，不允许跳过
- quality_tags 4-8 个
- negative_prompt 必须包含通用负面词 + 视频特有排除项 + 一致性专项
- **时代地域段不可省略**——从 era_setting / world_setting 提炼

### 7-B.3 连续性要求

- 同一场景的相邻片段，`scene_description` 中关键地标应一致（收银台位置、货架方向等）
- 同一角色的 `appearance_summary` 跨片段保持不变
- 前一片段的结束状态应在逻辑上衔接下一片段的开始动作
- 风格锚定中引用的 F01 元素在所有 segments 中保持一致
