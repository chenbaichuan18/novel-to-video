从小说文本中提取所有**重要人物**。每个获得独立 ID 的角色（包括有台词的无名角色）都将被记录，ID 将传递给后续人物提示词生成。对于多个相似的无名角色（如"客人甲"、"客人乙"），需通过唯一 ID 区分。

#### 字段说明

|字段名|含义|前端是否显示|备注|
|---|---|---|---|
|`id`|人物唯一标识，如 `char_1`|是（用于关联）|内部生成，前端可用于请求后续提示词|
|`name`|人物主要名称（全名或特征+编号）|是|有名字用名字，无名用"特征+编号"|
|`aliases`|别称、绰号或其他称呼数组|可选|如"小张"、"默哥"|
|`gender`|性别（英文枚举）：male=男 / female=女|是|必须推理，不可为unknown。使用英文枚举值以兼容F06输入期望|
|`age`|年龄或年龄范围（如"25岁"、"35-40岁"、"中年"）|是|必须推理，不可为"未知"|

|`clothing`|服装描述摘要（如"深蓝色工装"）|是|若原文无描述，根据角色职业和时代推理|
|`personality`|性格特征摘要（如"内向、安静"）|是|若原文无直接描写，根据言行推理|
|`role_type`|角色类型：`protagonist`/`antagonist`/`supporting`/`minor`|是|主角/反派/配角/临时有台词角色|

## 关键规则

### 1. 人物提取原则
- **有名称** → 必须提取，`name` 使用该名称。
- **无名称但有台词或与主角有对手戏** → 必须提取。`name` 使用 `特征 + 数字编号`（如"红裙女人1"、"戴帽子的男人"）。若多个同特征角色，用数字区分（"客人1"、"客人2"）。
- **无名称、无台词、无独立行为**（如"几个客人走进来"未再描述） → **不提取**。

## 2.字段规则

- **禁止空字段和"未知"**：`gender`、`age`、`appearance`、`clothing`、`personality` 必须填充具体值。若原文无直接信息，请根据角色的职业、时代、场景、类型惯例进行合理推断。
- **年龄推断示例**：学生→15-22岁；职场新人→22-26岁；资深职员→30-45岁；老人→60岁以上；若无任何线索，使用"25-35岁"作为默认范围。
- **外貌推断示例**：若无描写，根据职业给出默认外貌（如"便利店店员：普通面容，短发，略显疲惫"；"白领：整洁干练，化淡妆"）。
- **服装推断示例**：根据时代和职业给出合理服装（如"现代职场：衬衫/西装"；"便利店店员：工装"）。
- **性格推断示例**：根据角色的对话语气和动作推断（如"主动提问"→"直接、主动"；"自言自语"→"焦虑、内向"）。
- **`appearance_raw`** 可以为空字符串，但其他字段不得为空。

### 3. 推理指南

| 字段 | 推理依据 | 示例 |
|------|----------|------|
| `gender` | 代词（他→male, 她→female）、称谓（先生→male, 女士→female）、名字暗示、服装描述 | 出现"他" → male |
| `age` | 职业（学生→15-22；职场新人→22-26；资深→30-45；老人→60+）；对话提及（"老张"→40+）；无信息时默认"25-35岁" | 便利店店员 → 25-30岁 |
| `appearance` | 职业常规形象（警察→健壮；教师→斯文）；时代特征；若无则写"普通面容，体型匀称" | 程序员 → 戴眼镜，穿着随意 |
| `clothing` | 职业装（医生→白大褂；厨师→围裙）；时代（现代→T恤/衬衫）；季节暗示 | 冬天 → 穿大衣或羽绒服 |
| `personality` | 对话语气（主动提问→直接；自言自语→内向/焦虑）；动作描写（摔门→易怒） | 主动搭话 → 外向 |

### 4. 原文摘录（`appearance_raw`）
- 若原文有直接的外貌描写句子，原样摘录（保持标点）。
- 多个出处用中文分号 `；` 隔开。
- 若无任何原文描写，则留空字符串 `""`，**不要**写"无"或"暂无"。

### 5. 角色类型判定
- `protagonist`：全文视角主要跟随的人物，出现频率最高。
- `antagonist`：与主角对立的主要角色。
- `supporting`：重要配角，有名字或有多次出现。
- `minor`：临时出现但有台词或互动（如"服务员"、"快递员"）。

### 6. 输出格式
- 只输出纯 JSON 对象，不要包含任何解释性文字、markdown 代码块标记（如 ```json）。
- JSON 必须合法，无尾随逗号。
